Problem

Hot wallets are the single largest attack surface on Solana.

$50M+
drained from Solana hot wallets in the last 6 months

Every browser extension, every signing dApp, every malicious npm package is a path to your private key. The attack surface widens each time you open a new tab.

  • Malicious page silently re-signs a confirmed tx with a different recipient
  • Compromised npm dep ships ed25519 → captures the seed at runtime
  • Phishing clone of a wallet extension UI; user types the seed once, never again
  • Browser-level keylogger steals the password the moment it's typed

Existing answer · Hardware wallets

Better than hot. Still not enough.

Hardware wallets put the key in a secure element — but the channel between you and the secure element is still online and bidirectional.

  • USB / Bluetooth attack surface.Recent CVEs in Ledger Live, Trezor Suite, BIP39 implementations.
  • Hardware cost: $80 – $300.Vendor lock-in. Battery dependency. Firmware update discipline.
  • Blind signing on small screens.Complex Solana txs (Squads votes, DEX routes) don't fit on a Ledger screen.
  • Closed firmware.Most "hardware wallet security" claims live in code only the vendor can audit.

Aurora cold wallet (Breakout 2025) went hardware-only. Pollinet (Cypherpunk 2025) went Bluetooth mesh. Unruggable (Radar 2024) went MPC. None won a Grand Champion. The space is open.

The AirSign approach

Any device with airplane mode
becomes a cold-storage signer.

An old laptop. A phone in airplane mode. A Raspberry Pi with no Wi-Fi card. AirSign turns it into the device that holds your private key — and signs transactions over a one-way optical channel.

📡

1 · Prepare & Send

Online machine encrypts the unsigned tx (Argon2id + ChaCha20-Poly1305) and animates a fountain-coded QR stream.

🔐

2 · Air-gap Sign

Offline device scans, decrypts, signs with Ed25519, emits the signed response back as QR.

→
📨

3 · Receive & Broadcast

Online machine reassembles the response and broadcasts to devnet/testnet/mainnet via standard RPC.

Cryptographic stack

Every primitive named. None custom.

Audit-ready. RFC-cited. Constant-time by upstream design. The fountain layer carries no secret material — confidentiality lives in the AEAD layer above it.

RFC 8032
Ed25519 signing · Solana-native; FROST output is indistinguishable on chain
RFC 8439
ChaCha20-Poly1305 AEAD · per-session 96-bit random nonce; Poly1305 tag rejects tampering
RFC 9106
Argon2id KDF · OWASP 2024 minimum (m=64 MiB, t=3, p=1); Mainnet/Paranoid presets
RFC 9591
FROST threshold · t-of-n Ed25519, explicit binding, single 64-byte aggregated sig
Pedersen DKG
Distributed key generation · no trusted dealer; Schnorr proof-of-knowledge blocks last-participant bias
LT codes
Luby Transform fountain coding · survives ~30% frame loss; ChaCha8-seeded RNG for canonical decoding

Adoption story

We don't replace wallets.
We layer underneath them.

AirSign integrates as a wallet adapter. Existing dApps don't change; users opt into air-gap signing for high-value tx tiers. Distribution rides on Phantom and Squads — no separate app to install.

Tier 1 · Institutional custody

Custody platforms (Fireblocks-style) running Squads vaults. AirSign signs every multisig vote on a permanently offline device — no hardware fleet, no vendor lock-in.

Tier 2 · DAO treasuries

Squads v4 vault holders required to attest via AirSign. Treasury voting becomes air-gapped without buying everyone hardware wallets.

Tier 3 · Whales & high-value users

Phantom adds "Sign with AirSign" next to "Sign with Ledger". High-value tx flows through AirSign; daily flows stay in-extension.

Integration cost for Phantom or Squads: the @airsign/wallet-adapter package — under 200 LoC, MWA-compatible. We ship the adapter; they wire the toggle.
